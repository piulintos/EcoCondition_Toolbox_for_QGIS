EcoCondition Toolset

EcoCondition Toolset - A QGIS plugin for ecosystem condition assessment

This plugin contains a set of tools to prepared data for, and assess, the Ecosystem condition of any specific ecosystem type. The ecosystem condition assessment is based on the weighted sum of six specific ecosystem states (Physical, Chemical, Compositional, Structural, Functional, Landscape), following the system of environmental accounting developed by the United Nations.

This plugin allows for the assessment of the Ecosystem condition for a user ecosystem of choice, based on the weighted sum of variables related to 6 specific ecosystem states (Physical, Chemical, Compositional, Structural, Functional, and Landscape states).

Background

Ecosystem condition quantifies an ecosystem’s integrity, function and resilience. Under SEEA EA, it is operationalized by: (1) mapping consistent ecosystem units; (2) selecting structural, functional and compositional indicators; (3) defining reference bounds; and (4) aggregating indicators into a composite index. Comparing current and reference states reveals habitat trends, guides restoration and informs policy. Platforms like ARIES for SEEA automate forest condition accounts by integrating geospatial layers and applying SEEA EA protocols.

Indicator weights reflect intrinsic and service relevance, directional meaning, human sensitivity and data reliability. Methods range from simple sums and averages—which can mask interactions—to PCA or machine learning, which need training data and can reduce transparency, and fuzzy logic, which needs rule calibration. Weighted sums normalize each indicator, apply explicit weights and combine them, offering traceability, flexibility and GIS compatibility. To ensure robustness, normalization, collinearity checks and stakeholder input are essential.

The toolset

The toolset comprises the following tools:

1. Align layers (with clip and resample options),
2. Solve no-data issues (diff. sources & data formats),
3. Test data for multicollinearity,
4. Normalise and invert data (with no-data options),and
5. Calculate Ecosystem Condition.

Tool 1. Align layers

This tool allows the user to collect data from different sources, with different Coordinate Reference System (CRS), resolution and data types, and apply a set of operations on these layers (clip and resample), outputting a new set of layers, all with the same CRS, data type, number of columns and lines, and resolution.

Tool 2. Solve no-data issues

This tool allows the user to harmonise and to solve no-data values for different raster layers. It is beneficial to prepare raster layers from various sources and data types for subsequent calculations. It addresses potential issues with rasters in different formats, such as ‘Float32’, ‘Byte’, ‘UInt16’. The user can define a specific no-data value (default fvalue is -9999).

Tool 3. Test data for multicollinearity

This tool helps to detect and quantify multicollinearity between multiple raster layers. Multicollinearity refers to situations where one variable can be linearly predicted from others with a high degree of accuracy. The tool provides two complementary assessments: via a Spearman correlation; and via the Variance Inflation Factor (VIF).

Tool 4. Normalise and invert data

This tool allows the user to normalise the selected variables (typically to a 0–1 scale), and invert them when needed, ensuring comparability within each ecosystem state.

Tool 5. Calculate Ecosystem Condition

The ecosystem condition assessment tool enables the user to select the final layers for each ecosystem state, checks for layer alignment, and allows the user to assign weights to each ecosystem state and its related layers. The first values shown are default values, assuming that each state and its corresponding variable have the same weight.
